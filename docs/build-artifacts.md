# Build artifacts (`.build/` umbrella)

> Status: Active. Implements `openspec/changes/archive/2026-05-05-standardize-build-artifacts/` (archived).

## Layout

All runtime artifacts live under a single `.build/` directory at the repo root. One `.gitignore` line covers the whole tree.

| Subdir            | Lifetime             | CI behavior               | Local cleanup                                            |
| ----------------- | -------------------- | ------------------------- | -------------------------------------------------------- |
| `.build/cache/`   | reusable across runs | hash → CI cache key       | `npm run clean:cache` (fast iter without losing reports) |
| `.build/reports/` | per-run output       | always upload as artifact | `npm run clean:reports`                                  |
| `.build/dist/`    | per-deploy artifact  | upload to Pages           | always rebuilt                                           |

Sub-paths emitted today:

```
.build/cache/{astro,content,lhci,lychee,playwright,playwright-mcp}/
.build/reports/{playwright/{results,html},lhci,coverage}/
.build/dist/                 # astro build output, deployed to Pages
.build/.run.lock             # test-runner sentinel (hostname:pid:ts)
```

## SSoT contract

Every path string above is declared once, in `build.config.json`. Two consumer surfaces:

- **TS-aware tools** (`astro.config.ts`, `playwright.config.ts`, tests) `import { BUILD } from './build.config.ts'` and reference the typed constant. No string literals.
- **JSON/YAML/TOML tools** (Lighthouse CI, Lychee, GitHub Actions workflows) consume files generated by `scripts/sync-build-config.mjs` from the source-of-truth JSON.

`build.config.ts` is a typed wrapper:

```ts
import { readFileSync } from "node:fs";
import { fileURLToPath } from "node:url";
import { dirname, join } from "node:path";

const __dirname = dirname(fileURLToPath(import.meta.url));

export interface BuildPaths {
  /* … */
}
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object ? DeepReadonly<T[K]> : T[K];
};

const data = JSON.parse(
  readFileSync(join(__dirname, "build.config.json"), "utf8"),
) as BuildPaths;
const _validate: BuildPaths = data satisfies BuildPaths;
export const BUILD: DeepReadonly<BuildPaths> = data;
```

`BUILD` exposes string-typed leaves (NOT literal-string narrowing — `JSON.parse` widens; do not rely on `as const`-style narrowing in consumers).

## Generated files

`npm run sync:build-config` regenerates from `build.config.json` plus `scripts/fixtures/lhci-assertions.json`:

| Generated file      | Source                                                                                  |
| ------------------- | --------------------------------------------------------------------------------------- |
| `lighthouserc.json` | `build.config.json` (paths) + `scripts/fixtures/lhci-assertions.json` (assertion shape) |
| `lychee.toml`       | `build.config.json` (`cache_path`, `exclude_path`)                                      |

GitHub workflows are NOT regenerated wholesale today — they are hand-edited but the upload-artifact `path:` values point at `.build/...` paths. The Phase 4 generator (workflow YAML mutation via `yaml@2.5.x` Document API with stable `name:` selectors) is scaffolded but not yet wired into sync; that lands in a follow-up.

The sync generator emits a banner in every output:

- JSON: `"$generated": "build.config.json"` + `"$sha": "<16-char>"` sentinel keys.
- TOML/YAML: leading `# AUTOGENERATED — do not edit. Source: build.config.json (sha=<16-char>). Run: npm run sync:build-config` comment.

Manual edits to generated files are clobbered on the next sync. CI's drift gate (`npm run sync:build-config && git diff --exit-code`) catches stale generations.

## Add a tool — contributor checklist

When adding a new tool to the project (Vitest, Storybook, etc.), pick the path category:

1. **Does the tool produce cache?** Add a key under `cache` in `build.config.json` (e.g., `"vitest": ".build/cache/vitest"`).
2. **Does the tool produce reports?** Add a key under `reports` (e.g., `"vitest": ".build/reports/vitest"`).
3. **Does the tool produce a deploy artifact?** It probably doesn't — `dist` is for the deployed site only.
4. **Is the tool's config TS/JS-importable?** Edit its config to `import { BUILD } from './build.config.ts'`. No string literals.
5. **Is the tool's config JSON/YAML/TOML?** Add a generator branch in `scripts/sync-build-config.mjs`. Run `npm run sync:build-config` and commit both the SSoT diff (`build.config.json`) and the generated diff (the new config file).

Worked examples:

| Tool                     | Cache                                             | Reports                                   | Config strategy                        |
| ------------------------ | ------------------------------------------------- | ----------------------------------------- | -------------------------------------- |
| Astro                    | `BUILD.cache.astro`                               | —                                         | TS import in `astro.config.ts`         |
| Playwright               | `BUILD.cache.playwright` (env `PWTEST_CACHE_DIR`) | `BUILD.reports.playwright.{results,html}` | TS import in `playwright.config.ts`    |
| Lighthouse CI            | `BUILD.cache.lhci`                                | `BUILD.reports.lhci`                      | Generated `lighthouserc.json`          |
| Lychee                   | `BUILD.cache.lychee`                              | —                                         | Generated `lychee.toml`                |
| Vitest (hypothetical)    | `BUILD.cache.vitest`                              | `BUILD.reports.vitest`                    | TS import in `vitest.config.ts`        |
| Storybook (hypothetical) | `BUILD.cache.storybook`                           | —                                         | Generated `.storybook/main.ts` partial |
| Husky (no paths)         | —                                                 | —                                         | No `BUILD.*` entry; just install.      |

## Tree-sitter exclusion

Tree-sitter grammars are **explicitly excluded** from `BUILD.cache`. They are arch-specific binaries managed by the contributor's editor/IDE/global home directory (`~/.cache/tree-sitter/`). The `.gitignore` collapse drops `.tree-sitter/` from the project-level entry list.

## Clean scripts

```sh
npm run clean          # nuke .build/
npm run clean:cache    # nuke .build/cache/ only
npm run clean:reports  # nuke .build/reports/ only
```

All three route through `scripts/clean.mjs <target>` which:

- Validates the argv (`all`/`cache`/`reports`) — exit 64 on bad input.
- Reads `.build/.run.lock` (sentinel acquired by Playwright/LHCI on test start).
- Refuses to delete with exit 73 if the lock is held by a live PID on the same host.
- Auto-clears stale locks (dead PID OR mtime > 2 h OR foreign hostname).

Bare `rm -rf .build/...` is forbidden — it crashes mid-test runs.

## CI artifact retention

GitHub Actions workflow uploads `.build/reports/...` artifacts with `retention-days: 14` to bound storage growth against the 2 GB org artifact quota. Math: 50 MB Playwright report × 100 PRs × 14 d ≈ 70 GB-days, well within free quota.

## Generator-adjacent CODEOWNERS

The SSoT generator surface is dual-review-gated to close the workflow-injection vector — a single-reviewer rubber-stamp on `build.config.json` would otherwise bypass `.github/workflows/` review via the generator pathway. CODEOWNERS pairs `@artagon/owners` (admin team) with `@trumpyla` for:

- `build.config.{json,ts}`, `tsconfig.json`, `package.json`
- `scripts/sync-build-config.mjs`, `scripts/clean.mjs`
- `scripts/fixtures/lhci-assertions.json`
- `astro.config.ts`, `playwright.config.ts`, `lighthouserc.json`, `lychee.toml`
- `.github/workflows/`, `.github/dependabot.yml`, `.github/CODEOWNERS`
- `.gitignore`, `.gitattributes`, `.nvmrc`

Branch protection on `main` should enforce `required_approving_review_count: 2` + `require_code_owner_reviews: true`.

## Drift gate + bypass set

CI runs `npm run sync:build-config && git diff --exit-code`. Non-empty diff fails the build with `build config drift — run \`npm run sync:build-config\` locally and commit.`

The local pre-commit hook (Husky) runs the same check. Documented bypass set:

- `git commit --no-verify` — skips Husky; CI re-runs the same gate, so this only delays the failure.
- `[skip ci]` commit message — skips CI; monitored via a daily `gh api` workflow that flags PRs whose drift-gate status check is missing.
- Force-push / admin override — covered by branch protection settings.

## Bazel-readiness

`build.config.json` is the only data file Bazel ever needs to consume. Future `build_paths.bzl`:

```python
load("@bazel_skylib//lib:json.bzl", "json")
_data = json.decode(read_file("//:build.config.json"))
BUILD = struct(**_data)
```

The TS wrapper is a convenience for the Node ecosystem; the JSON file is the canonical source.

## Rollback

Per-step independent in reverse phase order (see `openspec/changes/archive/2026-05-05-standardize-build-artifacts/design.md` § Risks & Rollback). The deploy-affecting step (`actions/upload-pages-artifact@v4 path: ./.build/dist`) reverts via single workflow YAML edit.

If `astro.config.mjs` → `astro.config.ts` rename loses git blame post-squash-merge, document the squash exception or disable squash-merge on `main`.
